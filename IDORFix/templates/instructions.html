IDOR Demo Instructions

This demo shows an Insecure Direct Object Reference (IDOR) vulnerability that is resolved in a Patient Portal. Follow these steps to explore the vulnerability:

  • Log in with username: Alice Parker, password: pasthe$123.
  • View your profile at /profile/1.
  • Change the URL to /profile/2 to view Bob's profile.
  • Change the URL to /profile/3 to view Charlie's profile.
  • Each time you will get an error when you try to change to a different profile.
  • This is because this version is the fixed version.
  • If you want to play around with the version that has the vulnerability you can run IDOR instead.

Note: In real systems, IDs may be harder to guess, and such vulnerabilities can lead to serious data breaches.
